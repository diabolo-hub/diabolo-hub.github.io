mysql字符串 collate字符串大小写区分

发表于 2022-03-19 | 分类于 database ， mysql ， notes | 总阅读量 次
字数统计: 20k | 阅读时长 ≈ 20 mins.

背景

  1. 库中存在 test 和 TEST 两条数据，通过如下sql，均能查询出两条数据 大小写没有被区分查出

分析

mysql的表查询大小写是否敏感，取决于字段的collate设置
charset是utf8，而utf8的默认collate规则是utf8_general_ci。（可通过show charset命令查看。）
*_ci的含义是case insensitive，即不区分大小写

解决办法

  1. 查询条件前binary;
  2. 可以通过修改字段的collate为utf8_bin; (很多mysql不支持修改，具体的以实际执行结果为准)
    utf8_bin表示将字符串中的每一个字符用二进制数据存储，区分大小写。 其他修改方式：(https://www.cnblogs.com/yubin-huang/p/13601187.html)
