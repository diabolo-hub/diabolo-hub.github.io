depth-docker-excp-1-no-source-files-were-specified

发表于 2022-05-06
字数统计: 2.3k | 阅读时长 ≈ 2 mins.

背景

  1. Configurations配置
    Dockerfile: xxx/Dockerfile
    Context folder:
  2. Dockerfile
  3. 执行 run/debug 报错如下

no source files were specified

添加源文件失败。

分析

  1. 未对项目进行打包 package，未生产出 target/…/项目名称-版本号.jar
    在 idea Run/Debug Configurations里配置Dockerfile 路径的时候，Context folder路径指定为空，
    然后在Dockerfile中 ADD /.jar app.jar，结合来看是查找根目录下的 .jar，当然找不到，因为 *.jar 在target目录下
  2. 所以修改如下：
    configurations的Context folder配置为空不变，表示根目录，
    Dockerfile里的 ADD /.jar 调整为 ADD /target/.jar
    即可执行成功
