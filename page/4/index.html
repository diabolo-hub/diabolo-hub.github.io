ideal-插件-AceJump

发表于 2022-09-01
字数统计: 1.3k | 阅读时长 ≈ 1 mins.

跳转 - Default Mode：

Ctrl + ; 快捷键激活AceJump; mac输入 control + ;
在编辑器中输入想要到达位置部分字符，AceJump后搜索所有匹配的字符，然后在这些字符上打上标签
输入想要跳转位置黄色标签字符，光标就会定位到指定位置

导航功能[类似IDE中Navigate To] - Declaration Mode

此功能类似于IDE中Ctrl + b 快捷键
按两次 Ctrl + ; 快捷键激活该模式 ； mac输入 control + ;
输入部分想要定位字符datab 【目标database】
输入想要导航变量位置标签【g】

跳转 + 选中目标单词 - Target Mode

按3次 Ctrl + ; 快捷键
输入 想要选中单词部分或者全部 字符， 例如：datab
输入目标单词位置 标签，例如10行database： 输入 g 单词
结果跳转到10行database位置，且选中database

按行跳转 - Line Mode

a). 快捷键 Ctrl+Shift+;

该快捷键功能：会在 1) 行首、2) 行首第一个非空字符位置、3）行尾 三个位置加标签
b). 快捷键[只标记行首，行首第一个非空，行尾]： 1) 第一步：Ctrl + ; 2) 第二步 方向键： 向左方向键 - 标记行首，向右方向键 - 标记行尾 , 向上方向键 - 标记行首第一个非空字符

java-maven-maven-for-mac配置环境

发表于 2022-08-03
字数统计: 1.1k | 阅读时长 ≈ 1 mins.

配置

查看文件路径：1）首先，打开终端；

2）再将apache-maven-3.5.0 文件夹拖入到终端内，文件夹路径就会显示出来；

在终端打开配置环境变量到文件：

1）在终端输入 vim ~/.bash_profile，进入到环境变量配置文件里面；

2）进入后，是read模式，按下 i (编辑)键，进入insert模式；

3）将环境变量加入其实，环境变量如下：

export MAVEN_HOME=/Users/robbie/apache-maven-3.3.3

export PATH=$PATH:$MAVEN_HOME/bin

4）按下 ESC，退出insert模式；
5）输入 :wq (保存修改)退出当前文件；
6）使修改的环境变量bash_profile文件生效，输入 source .bash_profile，按下Enter键即可.
maven的安装包中conf文件夹下的setting.xml文件指定正确的仓库

如果仍然提示mvn找不到: zsh: command not found: mvn

输入:
➜ ~ mvn -v

提示:
zsh: command not found: mvn

解决方法:
step 1:

vim ~/.zshrc
step 2:(Add at the end of the file)

source ~/.bash_profile;
step 3:(Execution shell)

source ~/.bash_profile

Kafka常用命令

启动kafka（需要先启动zookeeper）

.\bin\windows\kafka-server-start.bat .\config\server.properties

后台常驻方式,带上参数 -daemon,如:

/kafka/bin/kafka-server-start.sh -daemon /kafka/config/server.properties

指定 JMX port 端口启动，指定 jmx，可以方便监控 Kafka 集群

JMX_PORT=9991 /kafka/bin/kafka-server-start.sh -daemon /kafka/config/server.properties

停止Kafka

/kafka/bin/kafka-server-stop.sh

创建topic

官方推荐如果kafka版本大于等于2.2使用–bootstrap-server替代–zookeeper （2.2以上也兼容–zookeeper）命令如下

kafka-topics.sh –create –bootstrap-server localhost:9092 –replication-factor 1 –partitions 1 –topic mytest

如果kafka版本小于2.2则命令如下

kafka-topics.sh –create –zookeeper localhost:2181 –replication-factor 1 –partitions 1 –topic mytest

Kafka本地测试

.\bin\windows\kafka-server-start.bat .\config\server.properties

创建主题，同一个主题创建一次即可。

.\bin\windows\kafka-topics.bat –create –zookeeper localhost:2181 –replication-factor 1 –partitions 1 –topic test1
或
.\bin\windows\kafka-topics.bat –create –bootstrap-server localhost:9092 –replication-factor 1 –partitions 1 –topic test1

启动生产者和消费者

.\bin\windows\kafka-console-producer.bat –broker-list localhost:9092 –topic test1

.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9092 –topic test1 –from-beginning

.\bin\windows\kafka-topics.bat –describe –bootstrap-server localhost:9092 –topic test1

.\bin\windows\kafka-topics.bat –bootstrap-server localhost:9092 –alter –topic test1 –partitions 5

消费消息

1.从头开始
.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9092 –topic test1 –from-beginning
2.从尾部开始
从尾部开始取数据，必需要指定分区：
.\bin\windows\kafka-console-consumer.bat –bootstrap-server localhost:9092 –topic test1 –offset latest –partition 0

消费者Group

  1. 指定Group
    .\bin\windows\kafka-consumer-groups.bat –bootstrap-server localhost:9092 –topic test1 -group test_group –from-beginning
