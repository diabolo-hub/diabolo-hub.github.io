no source files were specified

发表于 2021-04-12 | 分类于 depth ， docker ， excp
字数统计: 2.7k | 阅读时长 ≈ 3 mins.

背景

  1. Configurations配置
    Dockerfile: xxx/Dockerfile
    Context folder:
  2. Dockerfile
  3. 执行 run/debug 报错如下

no source files were specified

添加源文件失败。

分析

  1. 未对项目进行打包 package，未生产出 target/…/项目名称-版本号.jar
    在 idea Run/Debug Configurations里配置Dockerfile 路径的时候，Context folder路径指定为空，
    然后在Dockerfile中 ADD /.jar app.jar，结合来看是查找根目录下的 .jar，当然找不到，因为 *.jar 在target目录下
  2. 所以修改如下：
    configurations的Context folder配置为空不变，表示根目录，
    Dockerfile里的 ADD /.jar 调整为 ADD /target/.jar
    即可执行成功

Docker删除镜像和容器

一、删除容器
首先需要停止所有的容器（只停止单个时把后面的变量改为image id即可）
docker stop $(docker ps -a -q)
删除所有的容器（只删除单个时把后面的变量改为image id即可）
docker rm $(docker ps -a -q)
二、删除镜像
查看host中的镜像
docker images
删除指定id的镜像
docker rmi
想要删除untagged images，也就是那些id为的image的话可以用
docker rmi $(docker images | grep “^” | awk “{print $3}”)
删除全部的images
docker rmi $(docker images -q)
