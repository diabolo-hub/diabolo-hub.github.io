业务架构图 ：

描述系统对用户提供了什么业务功能。用户视角或业务视角

【画图技巧】

  1. 通过不同颜色来标识业务状态：比如说哪些业务发展状态好，哪些问题比较多，哪些比较稳定，哪些竞争比较激烈等。
  2. 业务分组管理：将类似的业务放在一个分组里面展现，用虚线框或者相同背景将其标识出来。
  3. 区块对齐：为了美观，可以改变不同区块的长短大小进行对齐，让整体看起来更美观。

业务架构是什么

描述企业（产品）业务上如何创造价值、传递价值、获取价值

立项

发表于 2022-11-01 | 分类于 depth ， architecture
字数统计: 1.1k | 阅读时长 ≈ 1 mins.

什么是项目

为创造独特的产品、服务和成果而进行的临时性工作

哪些“工作”需要立项

符合项目定义的一切个人或团体工作，因此除了常见的
新建项目、滚动项目，这些工作也需要立项：
对系统功能模块较大规模的改造工作，如：对接中台、重构
非业务直接相关，为提升管理效能而进行的工作，如：新工单平台、新项目管理工具等

为什么要立项

通过立项流程，从公司组织层面确定：
➢ 是否做这个项目（业务目标、业务价值、可行性、投
入产出比…）
➢ 谁来做这个项目（IT负责方、业务参与方、供应商团
队<如有>…）
➢ 从哪里出多少钱做这个项目（预算/人力资源来源、
费用金额…）
➢ 怎么做这个项目（自研、采购：单一来源 / 招投标）

软件架构概念

发表于 2022-10-28 | 分类于 depth ， architecture
字数统计: 1k | 阅读时长 ≈ 1 mins.

软件架构概念: https://www.jianshu.com/p/7b82fb03ca34

组成派：

架构 = 组件 + 交互。

决策派

架构属于设计，但设计不一定属于架构。架构设计的决策，将对整体质量、并行开发、适应变化等方面有着重大影响，例如：

  • 模块如何划分
  • 每个模块的职责如何
  • 每个模块的接口如何定义
  • 模块间采用何种交互机制
  • 开发技术如何选型
  • 如何满足约束和质量属性的需求
  • 如何适应可能发生的变化
