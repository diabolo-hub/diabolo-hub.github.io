自动标签（tags）

hexo-enhancer的做法是这样，启动后扫描_config.yml中的keywords与tags属性收集全局标签候选词，然后给每一篇文章自动扫描tags。
例如，_config.yml配置的标签可以是这样的:


hexo-enhancer会扫描您的.md文章，如果文章内出现以上关键词，则自动将其分配如文章的tags属性中。
需要特别指出的是，keywords是标准配置，它最终会出现在网页meta中，而tags是自定义的属性，只会被本插件使用。

next -> next6

hexo-3.9.0 -> hexo-4.2

  1. 卸载原有的hexo
    进入项目根目录，hexo uninstall hexo
  2. 查找hexo和nodejs匹配的版本
    hexo安装说明
    |—-|—-|
    Hexo 版本 最低兼容 Node.js 版本
    6.0+ 12.13.0
    5.0+ 10.13.0
    4.1 - 4.2 8.10
    4.0 8.6
    3.3 - 3.9 6.9
    3.2 - 3.3 0.12
    3.0 - 3.1 0.10 or iojs
    0.0.1 - 2.8 0.10
  3. 执行安装指定版本
    npm install hexo@4.2
  4. 安装过程

hexo g 异常

分析

  1. 我们发现hexo已经升级到4.2.1
  2. 从 git@github.com:theme-next/hexo-theme-next.git 克隆下来的next6
    只能是next6有问题，发现_config.yml中没有next的版本号，于是从github上单独下载next6的内容:
    https://github.com/theme-next/hexo-theme-next/releases/tag/v6.0.3
  3. 再次 hexo g，正常。。。
