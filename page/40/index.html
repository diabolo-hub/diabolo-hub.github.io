git add . 和 git add -u 和 git add -A 的区别

发表于 2018-05-26 | 分类于 git ， 笔记
字数统计: 849 | 阅读时长 ≈ 1 mins.

git add . ：他会监控工作区的状态树，使用它会把工作时的所有变化提交到暂存区，包括文件内容修改(modified)以及新文件(new)，但不包括被删除的文件。
git add -u ：他仅监控已经被add的文件（即tracked file），他会将被修改的文件提交到暂存区。add -u 不会提交新文件（untracked file）。（git add –update的缩写）
git add -A ：是上面两个功能的合集（git add –all的缩写）
git add -f App.class ：如果你确实想添加该文件，可以用-f强制添加到Git：（Use -f if you really want to add them.）

git配置全局和独立仓库的username和email配置

发表于 2018-02-13 | 分类于 git ， config
字数统计: 1.3k | 阅读时长 ≈ 1 mins.
  • Git配置全局和独立仓库的username和email配置：https://blog.csdn.net/u012733059/article/details/50804374?utm_source=blogxgwz4

估计很多新手开始使用git的时候都会配置一个全局的用户名和邮箱以便减少每建一个repository就要配置一次用户名和邮箱的麻烦，配置如下：
$ git config –global user.name ‘gitname’
$ git config –global user.email ‘github@xx.com’

如果你公司的项目是放在公司的github上面, 如果你不进行配置用户名和邮箱的话, 则默认会使用全局的, 这个时候就错误了, 正确的做法是针对公司的项目, 在公司项目的根目录下进行单独配置，配置如下：
$ git config user.name ‘gitname’
$ git config user.email ‘github@xx.com’

PS：最后可以使用git config –list 来查看当前配置，如果你在项目根目录下查看则是全局配置+当前项目的配置，如果只想查看全局配置的话可以在repository外输入此命令。使用的时候就是就近原则，优先使用当前目录下的配置。

git搭建多仓库配置

发表于 2018-02-13 | 分类于 git ， config
字数统计: 5.5k | 阅读时长 ≈ 5 mins.
  • Windows配置多个git用户：https://www.cnblogs.com/zengming/p/7908171.html
  • Git之同一台电脑连接多个远程仓库：https://blog.csdn.net/ManyPeng/article/details/77012468r

#两个仓库地址分别为：
git@github.com:YaleZhangdy/spring-practing.git
ssh://git@gitlab.ch-ins.com.cn:13022/group/writeoff_grp/ch-writeoff.git

1.检查本机是否有ssh key设置,切换到.ssh目录
$ cd ~/.ssh 或cd .ssh
如果有 id_rsa 和 id_rsa.pub，说明已存在一对密钥/公钥。

2.创建新的 密钥/公钥，并指定密钥名称，比如id_rsa_x（x为任意名称）
注：生成id_rsa私钥公钥时需要命不同文件名，密码可设可不设。
ssh-keygen -t rsa -f ~/.ssh/id_rsa_x -C “zhangdaiyin.cool@163.com”
操作完成后，该目录会多出 id_rsa_x 和 id_rsa_x.pub 两个文件。

3.添加ssh key到对应的GitHub,GitLab上面
登录GitHub系统；点击右上角账号头像的“▼”→Settings→SSH keys→Add SSH key。
复制id_rsa.pub的公钥内容到Key填写框内。

4.在 ~/.ssh/ 文件夹下创建一个 config 文件
$ touch config
$ vim config

5.在.ssh下面建立配置文件，文件名为config的文件且不需要添加文件后缀。
config文件配置内容：

第一个账号，默认使用的账号

第二个账号

参数描述：
HostName 　　　　　　　 　　真实的域名地址
IdentityFile 　　　　　　　 　　id_rsa的地址
PreferredAuthentications　　　 配置登录时用什么权限认证–可设为publickey,password publickey,keyboard-interactive等
User 　　　　　　　　　　　 配置使用用户名

可以通过 ssh-add -l 来确私钥列表 是L，不是1

$ ssh-add -l

可以通过 ssh-add -D 来清空私钥列表

$ ssh-add -D

6.查看SSH 密钥的值（完全复制），分别添加到对应的 GitHub 账户中
$ cat id_rsa.pub
$ cat id_rsa_x.pub
把这两个值分别 copy 到 GitHub 账号中的 SSH keys 中保存。

7.清空本地的 SSH 缓存，添加新的 SSH 密钥 到 SSH agent中
$ ssh-add -D
$ ssh-add id_rsa
$ ssh-add id_rsa_x
最后确认一下新密钥已经添加成功
ssh-add -l
$ ssh-add -l
2048 SHA256:md2u3nKLoLLB7vNWfeTDZzIZe1yCY4LjUibHQ3rquGE id_rsa (RSA)
2048 SHA256:SjHLErsOjuDEUpnZM9QRdKslCp1xsrE5vgkeFuRTGoQ id_rsa_x (RSA)

8.测试 ssh 链接
$ ssh -T git@gitlab.ch-ins.com.cn
Welcome to GitLab, @blank113!
$ ssh -T git@github.com
Hi YaleZhangdy! You’ve successfully authenticated, but GitHub does not provide shell access.

9.若执行ssh-add /path/to/xxx.pem是出现这个错误:Could not open a connection to your authentication agent，
则先执行如下命令即可：（来源：https://www.cnblogs.com/sheldonxu/archive/2012/09/17/2688281.html）
ssh-agent bash

10.取消 git 全局用户名/邮箱的设置，设置独立的 用户名/邮箱

取消全局 用户名/邮箱 配置

$ git config –global –unset user.name
$ git config –global –unset user.email

进入项目文件夹（git clone项目之后的文件夹），单独设置每个repo 用户名/邮箱

$ git config user.email “xxxx@xx.com”
$ git config user.name “xxxx”
