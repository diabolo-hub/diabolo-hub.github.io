virtualBox与本地挂载共享文件夹

发表于 2021-06-12 | 分类于 tool ， linux ， vm | 总阅读量 次
字数统计: 3.9k | 阅读时长 ≈ 4 mins.

参考: VirtualBox 安装增强功能 ubuntu server版
VirtualBox mount报错：/sbin/mount.vboxsf: mounting failed with the error: No such device

背景

  1. virtualBox虚拟机，Ubuntu Server服务器，win10专业版（本地）
  2. 通过虚拟机的”设备“去安装增强功能一直无法成功，不论是安装增强，还是”溢出虚拟盘“
  3. 所以考虑通过命令的形式执行增强挂载（共享文件夹）

步骤

  1. 确保当前使用的账号有root权限，即可以执行sudo
  2. 更新apt-get，请执行 sudo apt-get update，以及 sudo apt-get upgrade
  3. 安装依赖工具，sudo apt-get install dkms， sudo apt-get install build-essential
  4. 重启， sudo reboot now，如果命令无效或有执行错误，也可以使用外面的virtualbox的重启按钮
  5. 挂载cdrom， sudo mount /dev/cdrom /mnt/
  6. 执行安装命令，sudo /mnt/VBoxLinuxAdditions.run（挂载之后自动生成的文件）， 不同版本的.run文件名可能有区别
  7. 此时，我们需要 设备->共享文件夹->共享文件夹，请添加一个共享文件夹，一般会勾选自动挂载和固定分配两项，
    共享文件夹路径以及名称，最好都不要出现中文。
    请记下共享文件夹名称，且在共享文件夹中放入一个test.txt作为测试使用。
    截图如下:

  8. 执行共享文件夹的挂载命令：
    sudo mount -t vboxsf shared /home/share/
    shared，为共享文件夹界面定义的共享文件夹名称
    /home/share/，为Ubuntu中共享文件夹路径，需提前创建；当然该路径可以填/mnt/share，也许提前创建
  9. 当我们执行第7步时，报错如下：
    /sbin/mount.vboxsf: mounting failed with the error: No such device
    解决办法：

    再次执行第7步，即可成功

  10. 然后再/home/share/文件夹下即可找到本地电脑workspace-shared文件夹中的test.txt
  11. 卸载， sudo umount /mnt/ (卸载命令未测试）(安装命令，sudo /mnt/VBoxLinuxAdditions.run)

重启电脑后，挂载需要重新执行(挂载需要时间，可以等待，不用执行下面的命令)

sudo mount /dev/cdrom /mnt/

  1. sudo /mnt/VBoxLinuxAdditions.run
  2. ls
  3. sudo mount -t vboxsf shared /home/share/
